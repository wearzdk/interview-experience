### 《参考解析》

1. **线程池配置**：核心参数为corePoolSize, maxPoolSize, keepAliveTime等。CPU密集型配置N+1，IO密集型配置2N。规范包括严禁Executors静态工厂创建，必须使用ThreadPoolExecutor自定义。
2. **内存溢出处理**：及时止损需重启或扩容，通过jmap导出dump，使用MAT分析Leak Suspects。GC Roots包括虚拟机栈中引用的对象、方法区中类静态属性引用的对象等。
3. **MySQL分布式锁**：利用唯一索引行锁（INSERT）或通过`GET_LOCK()`函数实现，需注意锁超时、持有者自动释放及集群主从切换带来的锁失效问题。
4. **多级缓存一致性**：核心采用“先更新库，后删缓存”策略，配合延迟双删、订阅Binlog（如Canal）进行异步同步，保证数据的最终一致性。
5. **慢查询优化**：通过Explain分析执行计划，关注type、key、rows。处理方式包括添加索引、覆盖索引优化、调整SQL结构、拆分长事务。
