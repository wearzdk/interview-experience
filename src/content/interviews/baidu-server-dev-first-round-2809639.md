### 《参考解析》

1. **Redis SETNX锁底层**：SETNX利用Redis单线程特性实现互斥，通常配合SET key value NX PX expiration实现分布式锁，防止死锁。底层通过Redis字符串类型实现。

2. **500错误排查**：500错误通常是服务器端代码异常。排查思路：先查看Nginx访问日志定位请求，再通过cat/grep或tail -f查看业务日志中的Stack Trace，重点检查空指针、数据库连接失败或外部依赖超时。

3. **微博关注表设计**：关注表应包含 `id`, `follower_id` (关注者), `following_id` (被关注者), `create_time`。重点在于建立 `(follower_id, following_id)` 联合索引以支持查询关注列表，同时需考虑约束以防止重复关注。

4. **Redis缓存策略**：通常根据数据访问热度（高频读）、数据实时性要求及对数据库的压力进行判定。需注意缓存击穿、穿透及雪崩的防范措施。
