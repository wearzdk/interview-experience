### 《面试题目》
1. 做自我介绍。
2. 用 AI 编程工具写过的最大项目是什么，大概有多少行代码。
3. 使用 AI 编程时是否遇到过幻觉问题，如何处理。
4. 是否遇到过多个 Agent 同时修改同一个文件导致代码冲突或文件被写坏。
5. 介绍上一段实习中具体做了什么。
6. 结合薪资、岗位内容和团队情况，判断是否继续推进。

---

### 《参考解析》
1. **AI 项目规模回答**：可以从代码量、模块数量、功能复杂度、技术栈和上线状态说明规模，同时强调自己做了需求拆分、架构约束、测试和人工验收，而不是完全依赖模型输出。
2. **AI 幻觉处理**：常见办法包括给清晰上下文、限定改动范围、要求模型引用现有接口、先写测试再实现、用 lint/typecheck/build 验证，以及对关键逻辑人工 review。
3. **多 Agent 冲突处理**：多人或多 Agent 并行开发时，应拆分文件边界和任务边界，使用 Git 分支、代码 review 和小步提交。对同一文件的并发修改要先合并上下文，避免两个执行者互相覆盖。
4. **低薪小厂判断**：技术面之外也要关注薪资、成长空间、岗位真实性和团队管理成熟度。面试复盘时可以把它当作练手，但不必为不匹配的机会消耗太多时间。
