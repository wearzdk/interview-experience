### 《参考解析》

1. **Redis去重**：通常使用 `SETNX` 或布隆过滤器。模板ID需保证业务全局唯一，避免不同配置冲突。建议使用 `业务前缀:模板ID:用户ID` 作为 Key。
2. **线程池动态调整**：核心线程数可以通过 `setCorePoolSize` 动态修改。拒绝策略包括 `AbortPolicy` (丢弃抛异常)、`CallerRunsPolicy` (调用者线程执行)、`DiscardPolicy` (丢弃)、`DiscardOldestPolicy` (丢弃最旧)。
3. **SQL执行顺序**：`FROM` -> `JOIN` -> `WHERE` -> `GROUP BY` -> `HAVING` -> `SELECT` -> `ORDER BY` -> `LIMIT`。
4. **回表**：当查询索引树（非聚簇索引）中不包含所有需要的字段时，需要通过索引中的主键ID去聚簇索引树中再次查询，即为回表。
5. **接口响应慢优化**：检查数据库索引、SQL执行计划；排查网络延迟、慢接口阻塞；使用 Redis 缓存热点数据；引入异步处理（MQ）；对复杂业务进行分表或削峰填谷。
