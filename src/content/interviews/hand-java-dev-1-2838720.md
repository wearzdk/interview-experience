### 《参考解析》

*   **大数据量导出与深度分页：** 避免使用`limit offset, size`，改用游标查询（`where id > last_id limit size`）或使用ES进行分页。确保索引覆盖，减少回表。
*   **HashMap底层：** 基于数组+链表/红黑树实现。JDK8在链表长度达到8且数组容量大于64时转为红黑树，提升查找效率（O(log n)）。
*   **Redis缓存一致性：** 推荐使用“延迟双删”或“先更新数据库，再删除缓存”。若对一致性要求极高，可使用Canal监听Binlog异步删除缓存。
*   **数据库慢查询：** 通过`slow_query_log`定位，使用`explain`分析SQL。关注`type`（最好达到ref/range级别）、`key`（是否命中索引）和`rows`（扫描行数）。
*   **线程池核心参数：** `corePoolSize`（核心线程数）、`maximumPoolSize`（最大线程数）、`keepAliveTime`（空闲存活时间）、`workQueue`（阻塞队列）、`handler`（拒绝策略）。
