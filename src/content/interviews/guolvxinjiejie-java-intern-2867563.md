### 《参考解析》

1. **MySQL 优化方向**：分四层回答：①索引优化（合理建联合索引、覆盖索引，避免索引失效场景）；②SQL 优化（避免 SELECT *、禁止函数作用在索引列、用 EXPLAIN 分析执行计划）；③表结构优化（字段类型精准、适度冗余换查询速度）；④架构优化（读写分离、分库分表、引入缓存层）。

2. **多线程应用场景**：常见场景：批量导入时多线程并行处理、定时任务异步分片执行、IO 密集型接口用线程池并行调用第三方服务、消息消费多线程并发处理。关键考点：线程池参数设置（核心线程数 = CPU核 × (1 + IO等待时间/CPU时间)）、共享资源加锁（synchronized / ReentrantLock）、结果收集用 CompletableFuture.allOf。

3. **实习面试应对技巧**：提前调研公司主营业务（旅游行业的订单、库存、支付是高频技术场景）；对薪资谈判要有底线意识；"年级排名"问题如实回答，并可补充"专注于项目实践"来弥补排名不突出的情况。
