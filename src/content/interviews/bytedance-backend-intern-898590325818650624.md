### 《参考解析》

1. **SSE 与 WebSocket 的区别**：SSE（Server-Sent Events）基于 HTTP 长连接，是服务端到客户端的单向推送，实现简单、自带断线重连，适合日志推送、AI 流式输出等场景；WebSocket 是全双工协议，客户端和服务端可以双向实时通信，适合聊天、实时协作等需要频繁双向交互的场景，但实现和维护成本相对更高。
2. **Redis 单线程为什么还能支持高并发**：核心在于 I/O 多路复用（如 epoll）——单个线程通过事件循环同时监听多个客户端连接的读写事件，只在真正有数据可读写时才处理，避免了传统多线程模型下的线程切换开销和锁竞争；同时 Redis 的数据操作都在内存中完成，命令执行本身足够快，所以单线程也能达到很高的吞吐量。
3. **最长上升子序列（LIS）**：O(n²) 的动态规划解法是 `dp[i]` 表示以第 i 个元素结尾的最长上升子序列长度，`dp[i] = max(dp[j]) + 1`（对所有 `j<i` 且 `nums[j]<nums[i]`）；更优的 O(n log n) 解法维护一个 tails 数组，遍历元素时用二分查找找到第一个大于等于当前值的位置替换（或追加到末尾），tails 数组的长度即为 LIS 的长度。
4. **内存泄漏与 OOM 的区别**：内存泄漏是指对象已经不再被使用，但由于错误的引用关系导致 GC 无法回收，长期积累会耗尽内存；OOM（OutOfMemoryError）是内存泄漏积累到一定程度或单次分配的对象过大导致的最终结果——即内存泄漏是"因"，OOM 往往是"果"之一，但 OOM 也可能是正常业务下瞬时内存需求超过堆容量导致，不一定存在泄漏。
