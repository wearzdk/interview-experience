### 《参考解析》

1. **Graph RAG 与多跳检索**：传统 RAG 基于向量相似度做单次检索，容易漏掉需要多步推理才能关联起来的信息；Graph RAG 先把知识构建成知识图谱（实体+关系），检索时可以沿着图谱关系做多跳查询（Multi-hop），从而回答"A和B之间有什么关联"这类需要跨文档、跨实体推理的复杂问题，弥补了传统向量检索"只能找相似、不能做推理"的短板。
2. **SpringBoot/MyBatis 背后的 IOC/AOP**：SpringBoot 的自动装配本质上是 IOC 容器根据条件注解（`@ConditionalOnClass` 等）自动创建并注入 Bean，减少手动配置；MyBatis 则通过 AOP 式的动态代理（为 Mapper 接口生成代理对象）拦截方法调用，将其转换为对应的 SQL 执行，两者都是"控制反转+面向切面"思想在具体框架上的落地。
