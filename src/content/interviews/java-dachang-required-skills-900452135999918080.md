### 《面试题目》

1. 大厂 Java 后端面试真正在考察什么？（并非只是「会框架写CRUD+背八股」）
2. Java 语言与 JVM 底子够不够硬（一面主战场）
3. 主流工程栈能不能独立落地（Spring、MySQL、Redis、MQ）
4. 分布式与高并发有没有真实认知（二面、三面分水岭）
5. 项目能不能经得住深挖（所有轮次都会问）

---

### 《参考解析》

1. **一面主战场——Java/JVM 硬功底**：包括集合框架底层原理（HashMap扩容、红黑树化）、多线程并发模型（线程池、锁机制、AQS）、JVM 内存模型与垃圾回收算法，这是筛掉"只会调包"候选人的第一道关。
2. **二三面分水岭——分布式与高并发**：面试官更关心候选人是否有真实的高并发场景经验，如缓存穿透/雪崩/击穿的解决方案、分布式锁、限流降级、消息队列的可靠性保障，而不是停留在概念层面的背诵。
3. **项目深挖的应对策略**：项目中每一个技术选型都要能讲清楚"为什么这样做、有没有考虑过替代方案、遇到过什么坑"，只讲"用了什么技术"而讲不出设计权衡，很容易在深挖环节暴露出项目理解不够深入。
